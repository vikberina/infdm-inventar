# Wie war der Versionierungsprozess mit Git für Sie?
Obwohl die Einarbeitung anfänglich mit einer gewissen Eingewöhnungszeit verbunden war, stellte sich nach wenigen Schritten ein Gefühl von Effizienz ein.
# Welchen Vorteil bietet das Arbeiten auf einem extra Branch bei der Datenbereinigung?
Man kann mehrere verschiedene Versionen erstellen. Das ermöglicht, Fehler zu machen.
# Warum ist es sinnvoll, Korrekturen in separaten Commits statt in einem großen Commit zu speichern?
Korrekturen in separaten Commits erleichtern Teamarbeit. Separate Commits pro Spalte gewährleisten Transparenz und Rückverfolgbarkeit der Änderungen. Sollte ein Fehler in einer bestimmten Korrektur identifiziert werden, kann dieser spezifische Commit zurückgenommen werden, ohne dass die anderen Korrekturen verloren gehen.
